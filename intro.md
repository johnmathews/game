This project is a game for my children. I am a data engineer and web developer and I have not made a game before.

It will run in a web browser. It will be hosted on my homeserver, and run as one of more docker compose services.

The game needs to be savable, so that it can be played over many sessions from different clients. It will have more than
1 player.

It will be a combination of isometric exploration, 2D platformer and other puzzles.

It needs to be flexible, modifiable, and I expect the project to grow and change as the game is tested, used and enjoyed,
and as my children use it and get older.

## Initial ideas for gameplay and scope

The game starts in a village (The village is called Spaarndam and is based on the village we live in.) and the player
seems an isometric top down view of the village. The player character can walk around the village and visit different
buildings. In the buildings they might see:

a. A "shop" where they have a menu and can choose what menu to buy b. A 2D plaform shooter level, where they have to
defeat an enemy, or pass some obstacles

The game should be quite open and extendable - the rough idea is to explore the world and rescue people from different
countries who might be stuck, or to overcome challenges in different cities. Part of the game progress will be receiving
parts of a map when you succeed in a mini-quest or challenge.
